# RNA-seq Snakemake Pipeline

A reproducible, end-to-end RNA-seq processing workflow built with
[Snakemake](https://snakemake.readthedocs.io/) and
[pixi](https://pixi.sh/) for environment management. It takes raw paired-end
FASTQ files and produces QC reports, STAR alignments, gene-level counts,
normalized coverage bigWig tracks, a count matrix, and DESeq2 differential
expression tables.

## Pipeline overview

```
raw FASTQ
   │  fastqc (raw)
   ▼
fastp  ──► fastqc (clean)
   │
   ▼
STAR align (sorted BAM + ReadsPerGene)   (02-read-alignment)
   │
   ├──► samtools index ─► megadepth → bigWig   (03-megadepth-bigwig)
   │
   ▼
merge per-sample counts → count matrix   (04-quant)
   │
   ▼
DESeq2 (each group vs reference)         (05-deseq2)
   │
   ▼  (opt-in; runs on SYSTEM R, see below)
GO/KEGG enrichment (per contrast)        (06-enrichment)
```

| Step | Rule | Tool | Output dir |
|------|------|------|------------|
| 1 | `fastqc_raw` | FastQC | `results/01-fastp-qc/{sample}/raw_fastqc` |
| 2 | `fastp` | fastp | `results/01-fastp-qc/{sample}` |
| 3 | `fastqc_clean` | FastQC | `results/01-fastp-qc/{sample}/clean_fastqc` |
| 4 | `star_align` | STAR | `results/02-read-alignment` |
| 5 | `index_bam` | samtools | `results/02-read-alignment` |
| 6 | `megadepth_bigwig` | megadepth | `results/03-megadepth-bigwig` |
| 7 | `merge_counts` | python | `results/04-quant` |
| 8 | `deseq2` | DESeq2 | `results/05-deseq2` |
| _opt-in_ | `enrichment` | clusterProfiler (**system R**) | `results/06-enrichment/{contrast}` |

## Requirements

- [pixi](https://pixi.sh/) (handles all bioinformatics tool installation)
- Linux x86-64 (the `pixi.toml` pins `platforms = ["linux-64"]`)
- Enough RAM for the STAR index (~30 GB for a full human genome)

All tools are declared in `pixi.toml` and installed into isolated environments
on first use — you do **not** need to install STAR, R, etc. yourself.

## Setup

```bash
cd rnaseq

# Install all pixi environments (one-time; downloads tools)
pixi install
```

### Reference files

The pipeline expects pre-built references, configured in `config.yaml` under
`ref:`. They are **not** generated by this workflow:

- **STAR index** — a directory (config `ref.star_index`, e.g. `ref/star_index`)
  built once from the genome FASTA + GTF.
- **GTF annotation** — e.g. `ref/gencode.v41.annotation.gtf`.

Build the STAR index once with:

```bash
pixi run -e star STAR \
  --runMode genomeGenerate \
  --genomeDir ref/star_index \
  --genomeFastaFiles genome.fa \
  --sjdbGTFfile ref/gencode.v41.annotation.gtf \
  --sjdbOverhang 100 \
  --runThreadN 16
```

## Input: the sample sheet

Samples are described in `samples.tsv` (tab-separated). The shipped file is a
placeholder — replace it with your own:

```tsv
sample      group   layout  r1                              r2
sample_01   WT      PE      data/raw/sample_01_1.fastq.gz   data/raw/sample_01_2.fastq.gz
sample_02   WT      PE      data/raw/sample_02_1.fastq.gz   data/raw/sample_02_2.fastq.gz
sample_03   treatA  PE      data/raw/sample_03_1.fastq.gz   data/raw/sample_03_2.fastq.gz
sample_04   treatA  PE      data/raw/sample_04_1.fastq.gz   data/raw/sample_04_2.fastq.gz
sample_05   treatB  PE      data/raw/sample_05_1.fastq.gz   data/raw/sample_05_2.fastq.gz
sample_06   treatB  PE      data/raw/sample_06_1.fastq.gz   data/raw/sample_06_2.fastq.gz
```

| Column | Meaning |
|--------|---------|
| `sample` | Unique sample ID; used in all output paths and file names. |
| `group` | Condition/group label. Drives the DESeq2 comparisons (see below). |
| `layout` | Library layout. The pipeline currently assumes paired-end (`PE`). |
| `r1` / `r2` | Paths to the R1 / R2 FASTQ files (relative to the project dir). |

### Differential-expression comparisons

Groups are ordered by **first appearance** in `samples.tsv`. The first group is
the **reference (WT)**; every other group is contrasted against it:

- With groups `WT`, `treatA`, `treatB` (1, 2, 3), the pipeline runs
  **treatA vs WT** and **treatB vs WT** (i.e. 2 vs 1 and 3 vs 1).
- Add more groups and you automatically get more `*_vs_WT` comparisons.
- In each table a positive `log2FoldChange` means *up in the non-reference
  group* relative to WT.

To change the reference, just reorder `samples.tsv` so the desired group appears
first.

## Configuration

All tunable parameters live in `config.yaml`: reference paths, per-tool threads,
fastp options, STAR options (including `strandedness`, which selects the
`ReadsPerGene.out.tab` column used for counting), and the DESeq2 significance
thresholds.

**Strandedness** — set `star.strandedness` to one of:

| Value | ReadsPerGene column | Typical kit |
|-------|---------------------|-------------|
| `unstranded` | 2 | unstranded |
| `forward` | 3 | Ligation / dUTP-forward |
| `reverse` | 4 | Illumina TruSeq stranded (default) |

## Running

Snakemake itself runs in its own pixi environment; each rule then calls the tool
it needs via `pixi run -e <env> …`.

```bash
# Dry run — show what would execute
pixi run -e snakemake snakemake -n

# Run the full pipeline (adjust -j to your core count)
pixi run -e snakemake snakemake -j 32 -p

# Build a single target
pixi run -e snakemake snakemake -j 8 results/04-quant/counts.tsv

# Visualize the DAG
pixi run -e snakemake snakemake --dag | dot -Tsvg > dag.svg
```

Logs are written per rule under `logs/`.

## Enrichment analysis (optional, system R)

Step 06 runs GO and KEGG over-representation analysis on the differentially
expressed genes. Unlike every other step, it does **not** use pixi — it runs on
your **system R** with packages **you install yourself**. This keeps the heavy,
version-sensitive clusterProfiler + annotation stack out of the pixi
environments. It is **not** part of the default run; you trigger it explicitly.

**Supported organisms:** human, mouse, arabidopsis (only).

It consumes the DESeq2 tables from step 05 (`{contrast}.deseq2.tsv`), splits the
`significant` genes into up- and down-regulated sets by the sign of
`log2FoldChange`, and uses all tested genes as the background universe.

### 1. Install the required packages once (system R)

```bash
# Pick the OrgDb for your organism:
#   human -> org.Hs.eg.db   mouse -> org.Mm.eg.db   arabidopsis -> org.At.tair.db
Rscript -e 'if (!requireNamespace("BiocManager", quietly=TRUE)) install.packages("BiocManager"); \
  BiocManager::install(c("clusterProfiler", "AnnotationDbi", "ggplot2", "org.Hs.eg.db"))'
```

The script checks for these on startup and stops with the exact install command
if anything is missing. KEGG enrichment additionally needs **internet access**
(it queries the KEGG REST API); set `enrichment.run_kegg: false` to skip it.

### 2. Configure `config.yaml`

```yaml
enrichment:
  organism: human          # human | mouse | arabidopsis
  gene_id_type: ENSEMBL    # enrichGO keyType of your count-matrix IDs
  strip_version: true      # strip Ensembl ".N" suffix (GENCODE); false for TAIR/symbols
  rscript: "Rscript"       # system R; override with an absolute path to pin a specific R
  run_kegg: true           # KEGG needs internet
  pvalue_cutoff: 0.05
  qvalue_cutoff: 0.2
```

Defaults per organism: human/mouse → `gene_id_type: ENSEMBL`, `strip_version: true`
(KEGG genes are auto-converted to Entrez IDs); arabidopsis → `gene_id_type: TAIR`,
`strip_version: false` (TAIR loci used directly for both GO and KEGG).

### 3. Run it

```bash
# Builds results/06-enrichment/{contrast}/ for every contrast.
pixi run -e snakemake snakemake -j 8 enrichment
```

(Snakemake still launches from pixi; the rule itself calls your system
`Rscript`. The deseq2 tables are built first if they don't exist yet.)

## Key outputs

| File | Description |
|------|-------------|
| `results/01-fastp-qc/{sample}/{sample}_fastp.html` | fastp trimming report |
| `results/02-read-alignment/{sample}/{sample}.Aligned.sortedByCoord.out.bam` | STAR sorted alignment |
| `results/02-read-alignment/{sample}/{sample}.ReadsPerGene.out.tab` | STAR per-gene counts |
| `results/03-megadepth-bigwig/{sample}/{sample}.all.bw` | megadepth coverage track |
| `results/04-quant/counts.tsv` | gene × sample raw count matrix |
| `results/05-deseq2/normalized_counts.tsv` | size-factor normalized counts |
| `results/05-deseq2/{group}_vs_{WT}.deseq2.tsv` | DESeq2 results per comparison |
| `results/06-enrichment/{contrast}/go_{bp,cc,mf}_{up,down}.tsv` | GO enrichment per ontology and direction (opt-in) |
| `results/06-enrichment/{contrast}/kegg_{up,down}.tsv` | KEGG enrichment per direction (opt-in) |
| `results/06-enrichment/{contrast}/*_{barplot,dotplot}.pdf` | enrichment plots (only for non-empty results) |
| `results/06-enrichment/{contrast}/summary.txt` | per-contrast enrichment summary |

## Adapting to another organism

The defaults target human (GENCODE v41). To switch organism:

1. **Build a STAR index** for your genome + GTF.
2. **`config.yaml`** — point `ref.star_index` and `ref.gtf` at them, and adjust
   `star.sjdb_overhang` to `read_length - 1` if your reads differ from 2×101.

DESeq2 is organism-agnostic — no script edits are required. The optional
enrichment step (step 06), however, only supports **human, mouse and
arabidopsis**; set `enrichment.organism` accordingly (see above).

## Layout

```
rnaseq/
├── Snakefile          # workflow rules
├── config.yaml        # all parameters
├── samples.tsv        # sample sheet (edit this)
├── pixi.toml          # tool environments
└── scripts/
    ├── merge_star_counts.py    # STAR ReadsPerGene -> count matrix
    ├── snakemake_deseq2.R      # DESeq2 differential expression
    └── enrichment.R            # GO/KEGG enrichment (opt-in; system R)
```
