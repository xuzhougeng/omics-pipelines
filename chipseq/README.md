# ChIP-seq Snakemake Pipeline

A reproducible, end-to-end ChIP-seq processing workflow built with
[Snakemake](https://snakemake.readthedocs.io/) and
[pixi](https://pixi.sh/) for environment management. It takes raw paired-end
FASTQ files and produces QC reports, aligned/filtered BAMs, normalized bigWig
tracks, IP-vs-input ratio tracks, **input-controlled** peak calls, a consensus
peak set, a count matrix, and gene-level peak annotation.

It mirrors the sister `atacseq/` pipeline, with two ChIP-specific differences:

1. **Input/control samples.** `samples.tsv` has a `control` column that links
   each IP sample to its matched input. MACS3 calls peaks with `-c` against
   that input, and a `bamCompare` step emits IP/input log2-ratio tracks.
2. **No Tn5 shift.** BAM filtering keeps the ATAC-style quality filters
   (dedup, nuclear chroms, MAPQ, proper pairs) but drops the +4/−5 read shift.

## Pipeline overview

```
raw FASTQ
   │  fastqc (raw)
   ▼
fastp  ──► fastqc (clean)
   │
   ▼
bowtie2 align ─► samtools sort        (02-read-alignment)
   │
   ▼
sambamba markdup                      (03-mark-duplication)
   │
   ▼
filter BAM (MAPQ, nuclear chroms,     (04-bam-postprecessing)
proper pairs, dedup)
   │
   ├──► deepTools bamCoverage → bigWig          (05-deeptools-bigwig)
   ├──► deepTools bamCompare (IP/input) → bigWig (06-deeptools-bigwig-compare)
   │
   ▼
MACS3 callpeak  -t IP  -c input       (07-peak_calling)   [IP samples only]
   │
   ▼
merge → consensus peaks (bedtools)    (09-peak-quant)
   │
   ├──► featureCounts → counts.tsv
   └──► ChIPseeker → peaks_annotated.tsv
```

| Step | Rule | Tool | Output dir | Scope |
|------|------|------|------------|-------|
| 1 | `fastqc_raw` | FastQC | `results/01-fastp-qc/{sample}/raw_fastqc` | all |
| 2 | `fastp` | fastp | `results/01-fastp-qc/{sample}` | all |
| 3 | `fastqc_clean` | FastQC | `results/01-fastp-qc/{sample}/clean_fastqc` | all |
| 4 | `bowtie2_align` | bowtie2 + samtools | `results/02-read-alignment` | all |
| 5 | `sambamba_markdup` | sambamba | `results/03-mark-duplication` | all |
| 6 | `filter_bam` | pysam | `results/04-bam-postprecessing` | all |
| 7 | `bigwig` | deepTools bamCoverage | `results/05-deeptools-bigwig` | all |
| 8 | `bigwig_compare` | deepTools bamCompare | `results/06-deeptools-bigwig-compare` | **IP** |
| 9 | `macs3_callpeak` | MACS3 (`-c` input) | `results/07-peak_calling` | **IP** |
| 10 | `merge_peaks` | bedtools | `results/09-peak-quant` | — |
| 11 | `featurecounts` | subread | `results/09-peak-quant/featurecounts` | all |
| 12 | `chipseeker_annotate` | ChIPseeker | `results/09-peak-quant` | — |

## Requirements

- [pixi](https://pixi.sh/) (handles all bioinformatics tool installation)
- Linux x86-64 (the `pixi.toml` pins `platforms = ["linux-64"]`)

All tools are declared in `pixi.toml` and installed into isolated environments
on first use — you do **not** need to install bowtie2, MACS3, R, etc. yourself.

## Setup

```bash
cd chipseq

# Install all pixi environments (one-time; downloads tools)
pixi install
```

For the chipseeker environment, install the annotation packages manually once:

1. Enter the R console: `pixi run -e chipseeker R`
2. Paste and run:

```R
install.packages("BiocManager")
BiocManager::install(c("TxDb.Hsapiens.UCSC.hg19.knownGene", "org.Hs.eg.db", "GO.db"))
```

### Reference files

The pipeline expects pre-built references, configured in `config.yaml` under
`ref:`. They are **not** generated by this workflow:

- **bowtie2 index** — e.g. `ref/human.{1..4}.bt2`, `ref/human.rev.{1,2}.bt2`
  (config points at the prefix `ref/human`).
- **GTF annotation** — e.g. `ref/gencode.v41.annotation.gtf`.

Build the bowtie2 index once with:

```bash
pixi run -e default bowtie2-build genome.fa ref/human
```

## Input: the sample sheet

Samples are described in `samples.tsv` (tab-separated). The shipped file is a
placeholder — replace it with your own:

```tsv
sample           group  type    control          layout  r1                              r2
H3K27ac_WT_rep1  WT     IP      input_WT_rep1     PE      data/raw/...                    data/raw/...
H3K27ac_WT_rep2  WT     IP      input_WT_rep2     PE      ...                             ...
H3K27ac_KO_rep1  KO     IP      input_KO_rep1     PE      ...                             ...
H3K27ac_KO_rep2  KO     IP      input_KO_rep2     PE      ...                             ...
input_WT_rep1    WT     input                     PE      ...                             ...
input_WT_rep2    WT     input                     PE      ...                             ...
input_KO_rep1    KO     input                     PE      ...                             ...
input_KO_rep2    KO     input                     PE      ...                             ...
```

| Column | Meaning |
|--------|---------|
| `sample` | Unique sample ID; used in all output paths and file names. |
| `group` | Condition/group label (e.g. WT / KO). Useful for downstream comparisons. |
| `type` | `IP` or `input` — informational; the pipeline keys off `control`. |
| `control` | **The matched input sample's `sample` ID.** Leave empty for input rows. |
| `layout` | Library layout. The pipeline currently assumes paired-end (`PE`). |
| `r1` / `r2` | Paths to the R1 / R2 FASTQ files (relative to the project dir). |

### How input/control drives the workflow

- A row is treated as an **IP sample** when its `control` column is non-empty;
  rows with an empty `control` are **inputs**.
- **Every** sample (IP and input) goes through QC → alignment → markdup →
  filtering → per-sample coverage bigWig.
- **Only IP samples** get a `bamCompare` IP/input log2-ratio track and a MACS3
  peak call, where `-c` points at `results/.../{control}.filtered.bam`.
- The consensus peak set is the union of the IP peak calls; `featureCounts`
  quantifies it across **all** BAMs (inputs included, so you have background
  columns for normalization / differential binding).

Point several IP samples at the same input, or give each replicate its own
input — both work; the `control` value just names the sample to use.

## Configuration

All tunable parameters live in `config.yaml`: reference paths, per-tool threads,
fastp/bowtie2/MACS3 options, the bamCoverage/bamCompare bin sizes, the list of
nuclear chromosomes and MAPQ threshold used for BAM filtering, and the
ChIPseeker annotation settings.

**Narrow vs broad peaks.** The default calls narrow peaks (TFs, H3K4me3,
H3K27ac). For broad marks (H3K27me3, H3K36me3, …) add `--broad` to the MACS3
command and switch the output/target from `*_peaks.narrowPeak` to
`*_peaks.broadPeak`.

## Running

Snakemake itself runs in its own pixi environment; each rule then calls the tool
it needs via `pixi run -e <env> …`.

```bash
# Dry run — show what would execute
pixi run -e snakemake snakemake -n

# Run the full pipeline (adjust -j to your core count)
pixi run -e snakemake snakemake -j 32 -p

# Build a single target
pixi run -e snakemake snakemake -j 8 results/09-peak-quant/consensus_peaks.bed

# Visualize the DAG
pixi run -e snakemake snakemake --dag | dot -Tsvg > dag.svg
```

Logs are written per rule under `logs/`.

## Key outputs

| File | Description |
|------|-------------|
| `results/01-fastp-qc/{sample}/{sample}_fastp.html` | fastp trimming report |
| `results/04-bam-postprecessing/{sample}/{sample}.filtered.bam` | filtered BAM (dedup, MAPQ, nuclear, proper-pair) |
| `results/05-deeptools-bigwig/{sample}/{sample}.filtered.bw` | CPM-normalized coverage track |
| `results/06-deeptools-bigwig-compare/{ip}/{ip}.log2ratio.bw` | log2(IP/input) track |
| `results/07-peak_calling/{ip}/{ip}_peaks.narrowPeak` | per-IP MACS3 peaks (vs input) |
| `results/09-peak-quant/consensus_peaks.bed` | merged consensus peak set |
| `results/09-peak-quant/featurecounts/counts.tsv` | consensus-peak × sample count matrix |
| `results/09-peak-quant/peaks_annotated.tsv` | ChIPseeker gene annotation of peaks |

## Adapting to another organism

The defaults target human (hg38 / GENCODE v41). To switch organism:

1. **`config.yaml`**
   - `ref.bowtie2_index`, `ref.gtf` → your genome's index prefix and GTF.
   - `ref.macs_genome` → effective genome size (`mm` for mouse, or a number).
   - `filter_bam.nuclear_chroms` → that genome's chromosome names.
   - `chipseeker.organism` / `chipseeker.anno_db` → e.g. `"Mus musculus"` /
     `"org.Mm.eg.db"`.
2. **`pixi.toml`** — add the matching annotation package to the `chipseeker`
   feature (e.g. `bioconductor-org.mm.eg.db`), then `pixi install`.
