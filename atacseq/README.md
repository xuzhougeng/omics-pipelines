# ATAC-seq Snakemake Pipeline

A reproducible, end-to-end ATAC-seq processing workflow built with
[Snakemake](https://snakemake.readthedocs.io/) and
[pixi](https://pixi.sh/) for environment management. It takes raw paired-end
FASTQ files and produces QC reports, aligned/filtered BAMs, normalized bigWig
tracks, per-sample peak calls, a consensus peak set, a count matrix, and
gene-level peak annotation.

## Pipeline overview

```
raw FASTQ
   │  fastqc (raw)
   ▼
fastp  ──► fastqc (clean)
   │
   ▼
bowtie2 align ─► samtools sort        (02-read-alignment)
   │
   ▼
sambamba markdup                      (03-mark-duplication)
   │
   ▼
filter BAM (MAPQ, nuclear chroms,     (04-bam-postprecessing)
proper pairs) + Tn5 shift
   │
   ├──► deepTools bamCoverage → bigWig (05-deeptools-bigwig)
   │
   ▼
MACS3 callpeak (per sample)           (07-peak_calling)
   │
   ▼
merge → consensus peaks (bedtools)    (09-peak-quant)
   │
   ├──► featureCounts → counts.tsv
   └──► ChIPseeker → peaks_annotated.tsv
```

| Step | Rule | Tool | Output dir |
|------|------|------|------------|
| 1 | `fastqc_raw` | FastQC | `results/01-fastp-qc/{sample}/raw_fastqc` |
| 2 | `fastp` | fastp | `results/01-fastp-qc/{sample}` |
| 3 | `fastqc_clean` | FastQC | `results/01-fastp-qc/{sample}/clean_fastqc` |
| 4 | `bowtie2_align` | bowtie2 + samtools | `results/02-read-alignment` |
| 5 | `sambamba_markdup` | sambamba | `results/03-mark-duplication` |
| 6 | `filter_bam` | pysam | `results/04-bam-postprecessing` |
| 7 | `bigwig` | deepTools | `results/05-deeptools-bigwig` |
| 8 | `macs3_callpeak` | MACS3 | `results/07-peak_calling` |
| 9 | `merge_peaks` | bedtools | `results/09-peak-quant` |
| 10 | `featurecounts` | subread | `results/09-peak-quant/featurecounts` |
| 11 | `chipseeker_annotate` | ChIPseeker | `results/09-peak-quant` |

## Requirements

- [pixi](https://pixi.sh/) (handles all bioinformatics tool installation)
- Linux x86-64 (the `pixi.toml` pins `platforms = ["linux-64"]`)

All tools are declared in `pixi.toml` and installed into isolated environments
on first use — you do **not** need to install bowtie2, MACS3, R, etc. yourself.

## Setup

```bash
cd atacseq

# Install all pixi environments (one-time; downloads tools)
pixi install
```

For chipseeker environment, you should install packaged manully.

1. Enther R console by typing `pixi run -e chipseeker R`
2. Paste the following code and run

```R
install.packages("BiocManager")
BiocManager::install(c("TxDb.Hsapiens.UCSC.hg19.knownGene", "org.Hs.eg.db"))
BiocManager::install("GO.db")
BiocManager::install("org.Hs.eg.db")
```


### Reference files

The pipeline expects pre-built references, configured in `config.yaml` under
`ref:`. They are **not** generated by this workflow:

- **bowtie2 index** — e.g. `ref/human.{1..4}.bt2`, `ref/human.rev.{1,2}.bt2`
  (config points at the prefix `ref/human`).
- **GTF annotation** — e.g. `ref/gencode.v41.annotation.gtf`.

Build the bowtie2 index once with:

```bash
pixi run -e default bowtie2-build genome.fa ref/human
```

## Input: the sample sheet

Samples are described in `samples.tsv` (tab-separated). The shipped file is a
placeholder — replace it with your own:

```tsv
sample      group       layout  r1                              r2
sample_01   control     PE      data/raw/sample_01_1.fastq.gz   data/raw/sample_01_2.fastq.gz
sample_02   control     PE      data/raw/sample_02_1.fastq.gz   data/raw/sample_02_2.fastq.gz
sample_03   treatment   PE      data/raw/sample_03_1.fastq.gz   data/raw/sample_03_2.fastq.gz
sample_04   treatment   PE      data/raw/sample_04_1.fastq.gz   data/raw/sample_04_2.fastq.gz
```

| Column | Meaning |
|--------|---------|
| `sample` | Unique sample ID; used in all output paths and file names. |
| `group` | Condition/group label for downstream comparisons. |
| `layout` | Library layout. The pipeline currently assumes paired-end (`PE`). |
| `r1` / `r2` | Paths to the R1 / R2 FASTQ files (relative to the project dir). |

## Configuration

All tunable parameters live in `config.yaml`: reference paths, per-tool threads,
fastp/bowtie2/MACS3 options, the list of nuclear chromosomes and MAPQ threshold
used for BAM filtering, and the ChIPseeker annotation settings.

## Running

Snakemake itself runs in its own pixi environment; each rule then calls the tool
it needs via `pixi run -e <env> …`.

```bash
# Dry run — show what would execute
pixi run -e snakemake snakemake -n

# Run the full pipeline (adjust -j to your core count)
pixi run -e snakemake snakemake -j 32 -p

# Build a single target
pixi run -e snakemake snakemake -j 8 results/09-peak-quant/consensus_peaks.bed

# Visualize the DAG
pixi run -e snakemake snakemake --dag | dot -Tsvg > dag.svg
```

Logs are written per rule under `logs/`.

## Key outputs

| File | Description |
|------|-------------|
| `results/01-fastp-qc/{sample}/{sample}_fastp.html` | fastp trimming report |
| `results/04-bam-postprecessing/{sample}/{sample}.filtered.bam` | filtered BAM (dedup, MAPQ, nuclear, proper-pair) |
| `results/04-bam-postprecessing/{sample}/{sample}.filtered.shifted.bam` | Tn5-shifted BAM (+4 / −5) |
| `results/05-deeptools-bigwig/{sample}/{sample}.filtered.shifted.bw` | CPM-normalized coverage track |
| `results/07-peak_calling/{sample}/{sample}_peaks.narrowPeak` | per-sample MACS3 peaks |
| `results/09-peak-quant/consensus_peaks.bed` | merged consensus peak set |
| `results/09-peak-quant/featurecounts/counts.tsv` | consensus-peak × sample count matrix |
| `results/09-peak-quant/peaks_annotated.tsv` | ChIPseeker gene annotation of peaks |

## Adapting to another organism

The defaults target human (hg38 / GENCODE v41). To switch organism:

1. **`config.yaml`**
   - `ref.bowtie2_index`, `ref.gtf` → your genome's index prefix and GTF.
   - `ref.macs_genome` → effective genome size (`mm` for mouse, or a number).
   - `filter_bam.nuclear_chroms` → that genome's chromosome names.
   - `chipseeker.organism` / `chipseeker.anno_db` → e.g. `"Mus musculus"` /
     `"org.Mm.eg.db"`.
2. **`pixi.toml`** — add the matching annotation package to the `chipseeker`
   feature (e.g. `bioconductor-org.mm.eg.db`), then `pixi install`.

The ChIPseeker step reads organism, annotation DB, and TSS region from
`config.yaml`, so no script edits are required for a supported organism.
