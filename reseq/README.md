# Resequencing (DNA) Snakemake Pipeline

A reproducible workflow built with [Snakemake](https://snakemake.readthedocs.io/)
and [pixi](https://pixi.sh/) for environment management. It takes raw paired-end
FASTQ files and produces **analysis-ready BAMs**: adapter-trimmed, aligned with
`bwa mem` (read-group tagged), coordinate-sorted, and duplicate-marked.

Variant calling (VCF) is intentionally **out of scope** — the pipeline stops at
the analysis-ready BAM, which is the standard input for any downstream caller
(GATK, bcftools, DeepVariant, …).

## Pipeline overview

```
raw FASTQ (PE)
   │
   ▼
fastp  (adapter trim + QC report)        (01-fastp-qc)
   │
   ▼
bwa mem  (-R @RG)  ─►  samtools sort      (02-read-alignment)
   │
   ▼
sambamba markdup  (mark, reads kept)      (03-mark-duplication)
   │
   ▼
{sample}.bam  +  {sample}.bam.bai   ◄── analysis-ready
```

| Step | Rule | Tool | Output dir |
|------|------|------|------------|
| 1 | `fastp` | fastp | `results/01-fastp-qc/{sample}` |
| 2 | `bwa_mem` | bwa + samtools | `results/02-read-alignment/{sample}` |
| 3 | `sambamba_markdup` | sambamba | `results/03-mark-duplication/{sample}` |

## Requirements

- [pixi](https://pixi.sh/) (handles all bioinformatics tool installation)
- Linux x86-64 (the `pixi.toml` pins `platforms = ["linux-64"]`)

All tools are declared in `pixi.toml` and installed into isolated environments
on first use — you do **not** need to install bwa, samtools, etc. yourself.

## Setup

```bash
cd reseq

# Install all pixi environments (one-time; downloads tools)
pixi install
```

### Reference files

The pipeline expects a pre-built bwa index, configured in `config.yaml` under
`ref.bwa_index`. It is **not** generated by this workflow. The config value is
the index prefix, which is the genome FASTA path used when building the index:

```bash
# Build the bwa index once (creates ref/genome.fa.{amb,ann,bwt,pac,sa})
pixi run -e default bwa index ref/genome.fa
```

`config.yaml` then points at the prefix:

```yaml
ref:
  bwa_index: "ref/genome.fa"
```

## Input: the sample sheet

Samples are described in `samples.tsv` (tab-separated). The shipped file is a
placeholder — replace it with your own. Add one row per sample; all samples in
the sheet are processed.

```tsv
sample      group       layout  r1                              r2
sample_01   control     PE      data/raw/sample_01_1.fastq.gz   data/raw/sample_01_2.fastq.gz
sample_02   control     PE      data/raw/sample_02_1.fastq.gz   data/raw/sample_02_2.fastq.gz
sample_03   treatment   PE      data/raw/sample_03_1.fastq.gz   data/raw/sample_03_2.fastq.gz
sample_04   treatment   PE      data/raw/sample_04_1.fastq.gz   data/raw/sample_04_2.fastq.gz
```

| Column | Meaning |
|--------|---------|
| `sample` | Unique sample ID; used in all output paths, file names, and the `@RG` ID/SM/LB. |
| `group` | Condition/group label (carried for your own bookkeeping). |
| `layout` | Library layout. The pipeline assumes paired-end (`PE`). |
| `r1` / `r2` | Paths to the R1 / R2 FASTQ files (relative to the project dir). |

## Configuration

All tunable parameters live in `config.yaml`: the bwa index prefix, per-tool
threads, fastp trimming options, sambamba markdup buffer sizes, and the read
group platform. The `@RG` ID, SM, and LB are derived from the sample name; only
the platform (`read_group.platform`, default `ILLUMINA`) is set in config.

## Running

Snakemake itself runs in its own pixi environment; each rule then calls the tool
it needs via `pixi run -e default …`.

```bash
# Dry run — show what would execute
pixi run -e snakemake snakemake -n

# Run the full pipeline (adjust -j to your core count)
pixi run -e snakemake snakemake -j 32 -p

# Build a single sample's BAM
pixi run -e snakemake snakemake -j 8 \
  results/03-mark-duplication/sample_01/sample_01.bam.bai

# Visualize the DAG
pixi run -e snakemake snakemake --dag | dot -Tsvg > dag.svg
```

Logs are written per rule under `logs/`.

## Key outputs

| File | Description |
|------|-------------|
| `results/01-fastp-qc/{sample}/{sample}_fastp.html` | fastp trimming report |
| `results/02-read-alignment/{sample}/{sample}.bam` | coordinate-sorted alignment |
| `results/03-mark-duplication/{sample}/{sample}.bam` | analysis-ready BAM (dup-marked, RG-tagged) |
| `results/03-mark-duplication/{sample}/{sample}.bam.bai` | BAM index |

## Adapting to another organism

Point `ref.bwa_index` at the matching genome's index prefix and rebuild the
index with `bwa index`. Nothing else is organism-specific.
